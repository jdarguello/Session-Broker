# Introduction

**Session Broker** is a cloud-native identity bridge for AI agent orchestration. It connects channel identities (Slack/Teams), Keycloak authentication state, and workflow identities consumed by Agents and MCP servers.

## What it does

- **Caches JWTs after authentication callback** — receives direct Keycloak callback and stores encrypted token material in Redis
- **Resolves user identity on every chat event** — loads cached auth context by Slack user ID
- **Converts identity to workflow trust model** — maps user/channel identity into SPIFFE + Istio identity contract
- **Enforces token lifetime** — uses token `exp` as strict TTL for cache validity

## Core integrations

| Component | Role |
|---|---|
| **Dapr sidecar** | Service invocation transport and Redis state store abstraction |
| **Keycloak** | Identity provider and OAuth callback origin to the broker |
| **Redis** | Token cache (encrypted raw tokens + metadata, keyed by Slack user ID) |
| **Istio / SPIFFE** | Workflow identity target for downstream Agent/MCP access control |
| **Argo CD** | GitOps delivery — app-of-apps pattern drives all deployments |

## Primary workflows

### Write workflow (token cache)

1. User attempts privileged action in Slack/Teams.
2. External gateway initiates Keycloak auth with Google IdP.
3. Keycloak callback hits Session Broker directly.
4. Broker stores encrypted access/refresh/ID tokens in Redis with strict `exp` TTL and emits audit log.

### Read workflow (identity conversion)

1. A new chat event arrives with Slack identity.
2. Broker fetches token context by Slack user ID.
3. Broker validates expiry and authorization context.
4. Broker returns a workflow identity contract and mints/requests SPIFFE/Istio-compatible token for downstream agent execution.
