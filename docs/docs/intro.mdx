# Introduction

**Session Broker** is a cloud-native microservice for AI agent orchestration. It tracks user identities and session state across Agent and MCP (Model Context Protocol) invocations, acting as the authoritative source of truth for who is in a session and which agent owns it at any point in time.

## What it does

- **Creates sessions** — binds a Keycloak-authenticated user identity to a new agent session
- **Maintains state** — stores arbitrary session payload in Redis via the Dapr state store API
- **Hands off sessions** — transfers ownership of a session from one agent to another
- **Terminates sessions** — marks sessions as terminated when the interaction is complete

## Core integrations

| Component | Role |
|---|---|
| **Dapr sidecar** | Service invocation transport, Redis state store abstraction, JWT bearer middleware |
| **Keycloak** | Identity provider — issues JWTs validated by Dapr before requests reach the service |
| **Redis** | Persistent session store (accessed exclusively through the Dapr state API) |
| **Argo CD** | GitOps delivery — app-of-apps pattern drives all deployments |

## How authentication works

Dapr's `middleware.http.bearer` component intercepts every inbound HTTP request and validates the JWT signature and expiry against Keycloak's JWKS endpoint. Only requests carrying a valid token reach the FastAPI application. The service itself never calls Keycloak — it simply decodes the already-validated JWT payload to extract identity claims (`sub`, `email`, `realm_access.roles`).

```
Client ──Bearer JWT──▶ Dapr sidecar ──validates──▶ Keycloak JWKS
                              │
                              │ (valid token forwarded)
                              ▼
                       FastAPI app (decodes claims, no re-validation)
```
