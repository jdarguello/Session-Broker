# Introduction

**Session Broker** is a cloud-native identity bridge for AI agent orchestration. It currently provides authenticated session lifecycle APIs and is evolving toward full token brokerage for Agent/MCP identity resolution.

## Current capabilities

- **Creates and manages agent sessions** — exposes `POST/GET/PATCH/DELETE /sessions` plus `POST /sessions/{id}/handoff`
- **Extracts user identity from bearer token claims** — reads `sub`, `email`, and `realm_access.roles`
- **Stores session state in Redis via Dapr state API** — uses Dapr sidecar for state I/O abstraction
- **Supports health checks for runtime probes** — `GET /health`

## Token brokerage capabilities (implemented — demo)

- Token callback cache write: `POST /auth/callback/cache` — stores the user's token set keyed by Slack user ID
- Identity resolution: `POST /identity/resolve` — returns the user's access token plus identity claims so the caller can act as the user

## Core integrations

| Component | Role |
|---|---|
| **Dapr sidecar** | Service invocation transport and Redis state store abstraction |
| **Keycloak** | Identity provider; tokens are validated by Dapr bearer middleware |
| **Redis** | Session state persistence through Dapr state store (`redis`) |
| **Istio / SPIFFE** | Workload trust boundary enforced by EnterpriseClaw; outside this service's scope |
| **Argo CD** | GitOps delivery — app-of-apps pattern drives all deployments |

## Runtime workflows

### Current workflow (implemented)

1. Client calls Session Broker with `Authorization: Bearer <token>`.
2. Dapr bearer middleware validates signature/issuer/audience (cluster runtime).
3. Broker decodes JWT payload claims and creates/reads/updates session state.
4. Session data is persisted in Redis through Dapr state APIs.

### Token brokerage workflow (implemented — demo)

1. Keycloak callback stores the user's token set keyed by Slack user ID (`POST /auth/callback/cache`).
2. Authorized caller invokes `POST /identity/resolve` with the Slack user ID.
3. Broker returns the user's access token plus identity claims so the caller can act as the user.
