# Token Broker API

The token broker is the primary API surface of this service. It provides two operations:

1. **Write** — store encrypted token material after a Keycloak callback.
2. **Read** — resolve cached token data and return the user's access token plus identity claims to the caller.

## Security and policy

- Canonical cache key: Slack user ID (`slack:<slack_user_id>`).
- Tokens stored **AES-256-GCM encrypted** in Redis via Dapr.
- TTL derived strictly from the access token `exp` claim.
- Every read/write emits a structured audit log event. Raw token values are **never** returned in responses or logs.

---

## 1) Write — Keycloak callback cache

### REST

```
POST /auth/callback/cache
```

**Request body**

```json
{
  "slack_user_id": "U07ABC123",
  "access_token": "<keycloak-access-jwt>",
  "refresh_token": "<keycloak-refresh-token>",
  "id_token": "<keycloak-id-jwt>"
}
```

| Field | Required | Description |
|---|---|---|
| `slack_user_id` | ✅ | Primary cache key |
| `access_token` | ✅ | Keycloak access token (JWT). TTL is derived from its `exp` claim. |
| `refresh_token` | ✅ | Keycloak refresh token |
| `id_token` | ✅ | Keycloak ID token (JWT) |

**Response 200**

```json
{
  "cached": true,
  "slack_user_id": "U07ABC123"
}
```

**Response 422** — missing required fields.

### Dapr service invocation

```http
POST /v1.0/invoke/session-broker/method/auth/callback/cache
Content-Type: application/json
```

## 2) Read — resolve identity & return access token

### REST

```
POST /identity/resolve
```

**Request headers**

| Header | Required | Description |
|---|---|---|
| `X-Slack-User-Id` | ✅ | Slack user ID to look up |

No request body is required.

**Response 200 — authenticated**

```json
{
  "type": "authenticated",
  "sub": "9f8b1c2d-...",
  "email": "user@company.com",
  "roles": ["agent-user"],
  "slack_user_id": "U07ABC123",
  "access_token": "<keycloak-access-jwt>"
}
```

**Response 200 — unauthenticated** (no cached token found)

```json
{
  "type": "unauthenticated",
  "slack_user_id": "U07ABC123"
}
```

**Response 400** — `X-Slack-User-Id` header missing.

**Response 401** — cached token has expired; re-authentication required.

### Dapr service invocation

```http
POST /v1.0/invoke/session-broker/method/identity/resolve
X-Slack-User-Id: U07ABC123
```

## Notes

- Auth URL generation remains outside this broker.
- Broker does not own unauthenticated pre-session creation.
- Broker should never return raw cached tokens to callers.
