# Session Lifecycle API (Legacy)

This page documents the earlier session CRUD shape. The current primary contract for this project is the Token Broker API:

- [Token Broker API](./token-broker)

If session CRUD remains in scope for your implementation, keep these endpoints under `/sessions` as a secondary API surface while prioritizing:

1. `POST /auth/callback/cache`
2. `POST /identity/resolve`
