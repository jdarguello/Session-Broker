# Dapr + Keycloak Integration

Session Broker uses Keycloak as the authentication authority and Dapr as the runtime transport/state abstraction for callback and identity-resolution operations.

## How it works

### 1. Keycloak callback is the write trigger

After user authentication against Google IdP through Keycloak, Keycloak calls the broker directly. The broker persists encrypted token material in Redis keyed by Slack user ID.

Expected payload shape (conceptual):

```json
{
  "slack_user_id": "U07ABC123",
  "tokens": {
    "access_token": "...",
    "refresh_token": "...",
    "id_token": "..."
  },
  "claims": {
    "sub": "...",
    "realm_roles": ["authenticated"]
  }
}
```

### 2. Dapr state component handles Redis I/O

`gitops/session-broker/base/dapr-statestore.yaml` defines the state store used by the broker:

```yaml
apiVersion: dapr.io/v1alpha1
kind: Component
metadata:
  name: redis
  namespace: session-broker
spec:
  type: state.redis
  version: v1
  metadata:
    - name: redisHost
      value: "redis-master.redis.svc.cluster.local:6379"
```

### 3. Read path resolves channel identity to workflow identity

For each incoming chat event, broker resolves cache by Slack user ID and returns a workflow identity contract that downstream components can use in Istio/SPIFFE trust model.

Expected response shape (conceptual):

```json
{
  "authenticated": true,
  "authorization": {
    "allowed": true,
    "roles": ["authenticated", "agent:db.create"]
  },
  "workflow_identity": {
    "spiffe_id": "spiffe://enterpriseclaw.io/workflow/user/U07ABC123",
    "token": "<workflow-token-or-reference>"
  }
}
```

### 4. Dapr invocation support

The broker exposes regular REST endpoints and can be called through Dapr service invocation:

```http
POST /v1.0/invoke/session-broker/method/auth/callback/cache
POST /v1.0/invoke/session-broker/method/identity/resolve
```

## Keycloak realm setup

After Keycloak is deployed, configure the following (can be scripted via the Admin REST API):

| Setting | Value |
|---|---|
| Realm | `session-broker` |
| Client ID | `enterpriseclaw-channel-gateway` |
| Client scopes | Define action-specific scopes for agent/MCP access |
| Realm roles | At minimum `unauthenticated` and `authenticated`; extend with action roles |
| Callback target | Session Broker `POST /auth/callback/cache` |

Keycloak's JWKS endpoint will be available at:
```
http://keycloak.keycloak.svc.cluster.local/realms/session-broker/protocol/openid-connect/certs
```
